健身训练追踪网站 - 实施计划

 Context

 用户是一位健身爱好者，有私人教练。教练需要他每次训练完发一份"鉴定超负荷"模板（包含每个动作的组数、重量、次数、组间休息时间），用于追踪
 超负荷训练进度。

 现有 app 的问题：市面上几乎所有健身 app 把组间休息当成 notification 倒计时（预设 2
 分钟到了就响）。对健身高手不友好，因为高手依据"心率平稳"决定何时开始下一组，不是看时间。

 这个项目的核心理念：休息时间由用户体感决定（点开始/结束），不是倒计时；4 个变量（组数 / 重量 / 次数 /
 休息）就是全部；动作名完全自定义；训练完用户自己手机截图发教练，不在 app 里浏览历史；但底层用 localStorage 留住上次数据作为下次模板。

 项目名：Show Up（目录名 Show_Up）

 ---
 一、首页设计（基多拉三头入口）

 主视觉：项目根目录的 jiduola.png 是一张金色三头龙基多拉图（深色背景）。

 布局：
 - 全屏深色背景（与图片融为一体）
 - 图片居中显示
 - 用 CSS clickable area / SVG overlay 把图片分割成三个点击区（基多拉的三个头）：
   - 左头 → 肩 (Shoulders)
   - 中头 → 胸 (Chest)
   - 右头 → 背 (Back)
 - 三个 routine 固定，不可新增、不可改名
 - 点击对应头部 → 进入对应训练页
 - 视觉提示：hover 时对应头部有微妙的发光 / 缩放效果，让用户知道这里可点击

 技术实现：
 - 用 <img> + 绝对定位的 3 个透明 <button> 覆盖在三个头的位置
 - 按钮区域用百分比定位（适配不同屏幕尺寸）
 - 按钮 hover/active 时有轻微高亮（box-shadow / filter）

 全局训练计时器（顶部）

 - 进入 routine 那一刻开始计时（00:00:00）
 - 持续递增，直到用户点"结束训练"
 - 持久化：把"开始时间戳"存到 localStorage。每次进入页面，根据 now - startedAt 重新算 elapsed
   - 用户切出去再回来：继续显示真实流逝的时间
   - 用户刷新页面：继续显示
   - 用户手机关机：先不考虑（用户明确说不管）
 - 格式：HH:MM:SS（超过 1 小时也好读）

 添加动作流程

 - 用户点 + 添加动作
 - 弹出输入：[动作名] + [组数（数字）]
 - 确认后 → 生成新动作 + 对应行数的表格
 - 用户首次进入这个 routine 时（无历史），从这步开始
 - 用户后续可以无限添加动作

 第二次及之后进入同一个 routine

 - 自动加载上次完整数据（所有动作 + 所有组的 4 个变量）
 - 用户基于上次手动改（必须先点 Edit 才能改非休息字段）
 - 全局计时器重新开始（这是新的一次训练）

 ├─────┼───────────┼───────────────────────────────────────────┼───────────────────┤

 ├─────┼───────────┼───────────────────────────────────────────┼───────────────────┤

 ├─────┼───────────┼───────────────────────────────────────────┼───────────────────┤
 │ 3   │ 次数      │ 只读（Edit 模式下可改）                   │ 上次值 / 首次为 0 │
 ├─────┼───────────┼───────────────────────────────────────────┼───────────────────┤
 │ 4   │ 休息时间  │ 只读（Edit 模式下可改；计时器结果会覆盖） │ 上次值 / 首次为 0 │
 ├─────┼───────────┼───────────────────────────────────────────┼───────────────────┤
 │ 5   │ 计时按钮  │ 可点（仅当前组）                          │ 见下              │
 └─────┴───────────┴───────────────────────────────────────────┴───────────────────┘

 Edit 按钮（每个动作一个）

 - 位置：动作标题右侧
 - 平时：所有第 2/3/4 列是 只读文字，防止误触
 - 点 Edit → 这一个动作的所有 cell 变成 <input>，可手动改
 - 再点（变成 "Done" 或同一按钮文字切换）→ 退出编辑模式，回到只读
 - Edit 模式独立于计时：用户在 Edit 模式下也能点第 5 列计时（计时结果照常覆盖第 4 列）

 第 5 列计时按钮的状态机

 每行（组）的按钮状态由"当前组指针"决定。每个动作维护自己的"当前组指针"，初始指向第 1 组。

 - 当前组：按钮高亮可点
   - 初始文字：开始休息
   - 点击后：开始内部计时，按钮变 结束休息（颜色变化提示休息中）
   - 再点击：停止计时，把 elapsed + 10s 写入这一行的第 4 列，把当前组指针推进到下一行
 - 已完成组（指针之前的行）：按钮暗色不可点，显示 ✓ 或 "完成"
 - 未到组（指针之后的行）：按钮暗色不可点

 最后一组的特殊处理

 - 最后一组做完不需要休息（要换动作了）
 - 最后一组的第 5 列按钮：显示为 完成 或类似，点击不触发休息计时，只把这一组标记为完成
 - 最后一组的第 4 列：始终留空（导出/截图时为空 cell）

 四、休息按钮交互细节

 1. 用户在做第 N 组之前：第 N 组的按钮高亮显示「开始休息」
 2. 做完第 N 组 → 用户点「开始休息」
 3. 按钮立刻变成「结束休息」+ 颜色变成"休息中"主题色（如琥珀色）
 4. 同时第 4 列的数字变成实时跑动的计时器（mm:ss）作为视觉反馈
 5. 用户休息够了（心率平稳）→ 点「结束休息」
 6. 后台：finalRest = elapsedSeconds + 10，写入第 4 列
 7. 当前组指针推进到第 N+1 组（自动）
 8. 第 N+1 组的按钮亮起，第 N 组变成"已完成"暗色

 +10 秒 buffer 的理由：用户点了"结束休息"后还要躺回长椅、戴护腕、深呼吸再推杠铃，这段时间最少 10 秒。要给教练看到接近真实的休息时间。

 五、数据模型 (localStorage)

 // Key: 'showup:state'
 {
   // 三个固定 routine，key 是 'shoulders' | 'chest' | 'back'
   routines: {
     chest: {
       displayName: '胸',
       lastTrainedAt: 1717689600000,  // 上次完成训练的时间戳
       exercises: [  // 上次的数据，作为下次模板
         {
           name: '平板卧推',
           sets: [
             { weight: 50, reps: 10, restSeconds: 130 },
             { weight: 60, reps: 10, restSeconds: 185 },
             // ...
             { weight: 60, reps: 12, restSeconds: null }  // 最后一组 restSeconds 始终为 null
           ]
         }
       ]
     },
     shoulders: { ... },
     back: { ... }
   },
   // 当前进行中的训练（如果有）
   activeSession: {
     routineKey: 'chest',
     startedAt: 1717689600000,  // 进入 routine 那一刻的时间戳
     exercises: [ ... ],         // 训练中实时的数据
     currentSetCursors: {        // 每个动作的"当前组指针"
       0: 2,  // 第 0 个动作做到第 2 组
       1: 0   // 第 1 个动作还没开始
     }
   }
 }

 关键持久化时机：
 - 进入 routine 时：创建 activeSession，存 startedAt
 - 每次数据变化（输入、计时结束、添加动作）：写入 activeSession
 - 用户点"结束训练"：把 activeSession.exercises 拷贝到 routines[key].exercises，更新 lastTrainedAt，然后清空 activeSession
 - 如果 activeSession 存在且 routineKey 与当前进入的匹配：恢复进行中的训练（而不是从模板开始）

 七、技术栈

 - 纯 HTML + CSS + JavaScript（无框架）
 - Tailwind CSS via CDN：<script src="https://cdn.tailwindcss.com"></script>
 - 状态管理：单个全局 state 对象 + 一个 render() 函数（粗粒度重渲染，对这个规模够用）
 - 持久化：localStorage，每次 state mutation 后调一次 save()
 - 计时：setInterval(updateUI, 1000) 配合 Date.now() 算 elapsed（不是累加计数器，避免标签页背景时漂移）
 - 路由：用 window.location.hash（如 #chest），刷新可恢复当前页
 - 部署：本地双击 / Vercel / Cloudflare Pages / GitHub Pages

 八、UI / UX 细节

 1. 暗色主题：与基多拉图融合，健身房灯光强对眼睛友好，截图发微信也"专业"
 2. 大按钮：第 5 列计时按钮要大（汗手指好点）
 3. 休息中视觉：当前组所在的整行有暗色高亮 + 实时计时数字大且醒目
 4. 数字键盘：重量/次数 input 用 type="number" inputmode="decimal"
 5. 二次确认：「结束训练」「删除动作」加 confirm
 6. 紧凑布局：表格行高 ~36-40px，~20 组 2 张手机截图截完
 7. 首页极简：只有基多拉图，无任何文字 / 按钮 / 标题
 8. 路由记忆：刷新页面后回到刷新前的位置

 九、验证方式

 构建完成后端到端验证：

 1. 本地打开 index.html（双击 / python3 -m http.server）
 2. 首页测试：
   - 三个头能区分点击区域
   - 点击进入对应 routine
 3. 首次训练（胸）：
   - 进入胸训练 → 顶部计时器开始
   - 点 + 添加动作 → 输入"平板卧推" + 6 组
   - 第 1 组：点 Edit → 输入 50kg / 10 次 → 退出 Edit
   - 点第 1 组第 5 列「开始休息」→ 按钮变「结束休息」+ 实时计时
   - 等 1 分钟后点「结束休息」→ 第 4 列应显示 1:10（+10s buffer）
   - 当前组自动推进到第 2 组（第 2 组按钮亮起）
   - 重复至第 6 组（最后一组点完成，第 4 列空）
   - 加第二个动作"上斜推胸" 4 组，同样流程
   - 点「结束训练」→ 全局计时停止，数据保存
 4. 第二次训练（胸）：
   - 回首页 → 再点胸
   - 看到上次所有动作 + 数据预填，但都是只读
   - 顶部计时器从 00:00 开始
   - 点 Edit 改第 1 组重量 50 → 55 → 退 Edit
   - 用计时器跑一次，检查第 4 列被新值覆盖
 5. 持久化测试：
   - 训练中刷新页面 → 应该回到训练中状态，计时器继续，数据保留
   - 训练中关闭浏览器再开 → 同上
 6. 截图测试：
   - 训练完成后，用手机浏览器实际截图
   - 整个训练 ~20 组，应 2 张能截完
 7. 路由测试：
   - 在胸训练页刷新 → 还是胸训练页（不是首页）
